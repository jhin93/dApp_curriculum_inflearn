
지금껏 만든 부동산 컨트랙을 반라이브 식으로 배포.
진짜 에더를 내면서 하는 메인네트워크 말고, 테스트 네트워크에 배포를 해서 가짜 에더로 테스팅 진행.



배포 및 테스팅 순서

    1. 트러플 & 가나슈 로컬 네트워크
    2. 테스트 네트워크
    3. 메인 네트워크



테스트 네트워크

    1. 메인네트워크랑 기능적으로 똑같음
    2. 에더의 가치는 없음
       그래서 내 분산 어플리케이션이 네트워크에서 쓰일 때, 유저들에게 어떻게 보이며, 작동방식은 어떤지 알 수 있음.



컨트랙을 메인넷, 혹은 테스트넷에 배포할 수 있는 방법 3가지

    1. Geth
        장점: 내 로컬환경에서 직접 배포
        단점: 메인넷이나 테스트넷의 전체 블록을 미리 다운받아서 동기화시켜놔야 함.
        강의시점 기준 Ropsten 테스트넷의 블록이 360만개. 전체 다운받으려면 4~5시간 걸림. 
        매번 배포할때마다 블록을 동기화시켜야 함.

    2. 리믹스
        장점: 컨트랙 배포가 가장 빠르고 쉽게 가능. 배포를 위한 환경설정이 거의 필요하지 않음.
        단점: 리믹스 사이트 다운 or 없어짐의 긴장감

    3. 인퓨라
        사람들이 많이 사용한다고 함. 실제 메타마스크가 이더리움 노드에 연결하는 방식을 인퓨라를 사용함.
        장점: 유저들이 이더리움 노드 동기화할 필요없음. 인퓨라에서 제공하는 api 키를 사용해서 호스트된 이더리움 노드 클러스터에 연결하는 방식.
        단점: 서비스의 안정성 저하(maybe). 리믹스처럼 서비스를 갑자기 제공하지 않는다던가, api 키가 먹히지 않을 수 있음.

    강의는 리믹스와 인퓨라를 선택해서 진행됨.
